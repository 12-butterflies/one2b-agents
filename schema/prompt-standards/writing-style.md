# Writing Style Rules — One 2b / Jason Peter Stevens
**Locked:** 2026-06-08 | Updated: 2026-06-11
**Applies to:** All agents, all surfaces, all formats. No exceptions.

---

## Punctuation

**NO HYPHENS IN PROSE.**
Never use a hyphen as a prose punctuation device. Not in subject lines, not in WhatsApp messages, not in decks, not in emails, not in plain descriptive text.

**ALWAYS USE A COMMA.**
Wherever a hyphen might be tempting as prose punctuation, use a comma instead. If a comma does not fit grammatically, restructure the sentence until it does.

---

## CRITICAL EXCEPTION — compound technical identifiers retain their hyphens

The following identifiers are locked with their hyphens. Never strip them:

| Identifier | Why |
|---|---|
| `data-linked insurance products` | Required external product name — locked terminology |
| `BPC-157`, `TB-500`, `CJC-1295`, `AOD-9604`, `5-Amino-1MQ` | Peptide/compound names |
| `GHK-Cu`, `SS-31`, `MOTS-c` | Peptide/compound names |
| `claude-haiku-4-5`, `claude-sonnet-4-6`, `claude-opus-4-8`, `claude-fable-5` | Model IDs |
| ISO dates (`2026-06-11`), URLs, file paths, code identifiers | Standard formats |

**The rule in plain English:** Do not hyphenate English prose descriptions. Do not un-hyphenate locked technical compound identifiers.

---

## Correct examples

**Prose — hyphen removed:**
- Wrong: "first-of-its-kind platform"
- Right: "first of its kind platform"

- Wrong: "well-established methodology"
- Right: "well established methodology"

- Wrong: "risk-free yield"
- Right: "risk free yield" (and this phrase is banned anyway — see RED flags)

**Technical identifiers — hyphen kept:**
- Right: "data-linked insurance products" (product name — never "data linked insurance products")
- Right: "BPC-157 + TB-500 blend" (compound names)
- Right: "claude-sonnet-4-6" (model ID)
